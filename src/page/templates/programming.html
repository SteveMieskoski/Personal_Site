PROGRAMMING

I came into my self study of programming already having knowledge of programming albeit my prior experience mostly dealt with numerical applications. As such the two languages I already knew are matlab and mathematica (wolfram language) with some knwledge of fortran 90 I taught myself over a summer in college. I leveraged this knowledge and over the time I spent studying programming I have focused on two languages im particular. The first language I approached was Python due to it's general wide usage and it's usage in scientific and data analysis applications. I then followed this and learned javascript for reasons similar to the first reason I learned python. In addition to javascripts wide usage, I found the idea of creating a visual component an additional draw. Thus I'll begin with javascript.

I first approached javascript via node and angular, moving to angular 2 following the second release candidate. I found Angular 2's use of typescript with classes, imports, and the like more familiar and comfortable. Additionally, as typescript is related more to ES6 than ES5 I saw it having a dual purpose. One allowing me to learn a frontend framework and learning much of the new language elements introduced with ES6. I found many more as I progressed due to the requirement of converting typescript to javascript I was required to gain familiarity with build systems such as webpack, and gulp. This cross pollination extended beyond the client side code into my use of node as my server backend. Thus, I wrote the server code using ES5 syntax in the commonJS style. In essence using the MEAN stack.

I more recently moved from the MEAN stack, with angular2, to more traditional JS in the sense of using vanilla JS and jQuery. These two form the basis of the site you are now visiting, with the use of a couple of additional packages as well.
